##  组件注册
```text
概念:组件就是可复用的vue实例，在vue中万物皆组件
语法:Vue.component('name', {
      props:[],  (用来接受根组件传递给子组件的数据)
      template: 'html文本',
      data() {
        return {}
      },
      methods:{},
      computed:{}
    })
注:1）命名规范,js中驼峰命名,html中链接符-
   2）template中只能包含一个根节点
   3）组件与组件之间无法直接访问状态或方法,需要通过间接的组件通信来交流
   4）全局注册的行为必须在根组件创建之前发生。
   5）局部注册通过components来创建

处理组件之间无法直接访问的方案:
1）prop
2）事件
3）bus事件总线 bus.$on  bus.$emit
4）ref和$ref
   ref用来给元素或子组件注册引用信息,注册给元素,拿到的就是Dom节点;注册给子组件,拿到的就是组件对象
   $ref用来获取已注册的子组件信息
 
```

## 动态组件
```text
通过keep-alive元素将动态组件包裹起来,数据不会丢失
原理:第一次动态创建组件后会缓存起来,下次调用时再从内存中找出来
语法:
<keep-alive>
  <component :is="组件name"></component>
</keep-alive>
```

## 过渡
```text
diff算法:1）把虚拟Dom按照层级分解对比
        2）同key值进行对比
        3）同标签/组件对比

初始渲染过渡:
通过appear属性设置节点在初始渲染的过渡
多个元素过渡:
通过v-if/v-else来实现多个元素之间切换的过渡,并可以通过过渡模式来优化(in-out & out-in)
注:出现相同的标签名时需添加key属性来标记区分
多个组件过渡:
通过component动态组件来实现
列表过渡:
通过transition-group过渡组来实现,该元素会自动实例化一个标签,可以通过tag属性来设置标签类型
可复用的过渡:
将transition或transition-group作为根组件，然后将子组件放置在其中
```

## 生命周期
```text
beforeCreate  初始化事件和生命周期函数
created  初始化事件和生命周期函数之后(用于初始化状态或者挂到当前实例的一些属性)
beforeMount  页面载入前(完成了data和el数据初始化)
mounted  页面载入后html已经渲染(依赖于dom创建之后才进行初始化工作的插件、订阅bus.$on、ajax请求)
beforeUpdate  数据更新之前,每更新一次数据就会触发一次(用于记录数据更新之前的一些状态,位置等)
updated  数据更新之后,每更新一次数据就会触发一次(获取更新完成后的dom,才开始进行swiper工作的插件)
beforeDestroy  销毁之前(一般用于善后:清除计时器、清除绑定的事件等)
destroyed  销毁之后(与beforeDestroy类似)

$nextTick 在数据更新之后执行，并且只会执行一次
```

## 混入
```text
概念: 一个混入对象可以包含任意组件选项,当组件使用混入对象时,所有混入对象的选项会被“混合”进入该组件本身的选项内,如果有同名的选项会将数据进行合并,如果数据有冲突时以组件内部的数据为主
mixins:['obj']
```
